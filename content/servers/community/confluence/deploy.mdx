# Deploy on VMware Private AI Foundation

<DeployGuide>

<h2 id="prerequisites">Prerequisites</h2>

- VMware Private AI Foundation 2.x cluster
- Outbound HTTPS to your Confluence Cloud or Data Center instance
- A Confluence API token (Cloud) or PAT (DC)

<h2 id="container-image">Container / Image</h2>

```yaml
image: registry.example.com/community/mcp-confluence:0.3.0
```

<h2 id="secrets">Secrets</h2>

```bash
kubectl create secret generic mcp-confluence \
  --from-literal=CONFLUENCE_BASE_URL=https://example.atlassian.net \
  --from-literal=CONFLUENCE_API_TOKEN=...
```

<h2 id="networking">Networking</h2>

If your PAIF cluster egresses through a proxy, set `HTTPS_PROXY` and
`NO_PROXY` on the Deployment so MCP traffic stays inside the cluster while
Confluence requests use the proxy.

<h2 id="verification">Verification</h2>

```bash
kubectl exec -it deploy/mcp-confluence -- curl -s localhost:8080/healthz
```

Expected: `{"status":"ok"}`

<h2 id="troubleshooting">Troubleshooting</h2>

- **401 from Confluence** — Cloud tokens require email-as-username; DC PATs do not.
- **Slow page reads** — increase the container's CPU request; markdown conversion is CPU-bound.

</DeployGuide>
